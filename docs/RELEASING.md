# Releasing

Releases are produced by the **Release (Manual)** workflow in `.github/workflows/release.yml`. Trigger it from the GitHub Actions tab on github.com/grahamgilbert/limpet → Release (Manual) → **Run workflow**.

The workflow:

1. Reads the version from the `VERSION` file at the repo root.
2. Imports your Developer ID Application certificate into a temporary keychain.
3. Builds limpet in Release configuration with the Developer ID identity.
4. Notarizes the `.app` via `xcrun notarytool`.
5. Packages a DMG, signs it, notarizes it, and staples the ticket.
6. Creates a GitHub Release tagged `vX.Y.Z` with the DMG attached and an autogenerated changelog.

Bump `VERSION` before triggering the workflow.

## One-time setup — secrets

The workflow needs six repository secrets. Add them under **Settings → Secrets and variables → Actions → New repository secret**.

### `BUILD_CERTIFICATE_BASE64`

Your Developer ID Application certificate, exported from Keychain Access as a `.p12`, base64-encoded.

```sh
# 1. Open Keychain Access → My Certificates.
# 2. Right-click "Developer ID Application: Graham Gilbert (9D8XP85393)".
# 3. Export → save as DeveloperID.p12 with a password.
# 4. Base64-encode it for the secret value:
base64 -i DeveloperID.p12 | pbcopy
```

Paste the clipboard contents as the secret value.

### `P12_PASSWORD`

The password you set when exporting the `.p12`.

### `KEYCHAIN_PASSWORD`

Any random string. Used to lock the temporary keychain `apple-actions/import-codesign-certs` creates inside the runner. Never re-used outside the runner. Generate one with:

```sh
openssl rand -base64 24 | pbcopy
```

### `APPLE_ID`

The Apple ID email associated with your Apple Developer account.

### `APPLE_ID_PASSWORD`

An **app-specific password**, NOT your Apple ID password.

1. Sign in to [appleid.apple.com](https://appleid.apple.com).
2. Sign-In and Security → App-Specific Passwords → +.
3. Label it "limpet notarization", create.
4. Copy the resulting `xxxx-xxxx-xxxx-xxxx` string and paste as the secret value.

### `APPLE_TEAM_ID`

`9D8XP85393` (your Developer ID team identifier).

## Cutting a release

1. Edit `VERSION` (e.g. `0.2.0`).
2. Commit and push to `main`.
3. Go to **Actions → Release (Manual) → Run workflow → Run workflow**.
4. Wait ~10 minutes (notarization is the slow step).
5. The new release appears at github.com/grahamgilbert/limpet/releases.

## Troubleshooting

- **"errSecInternalComponent" or signing fails**: certificate didn't import. Verify `BUILD_CERTIFICATE_BASE64` decodes correctly and `P12_PASSWORD` matches.
- **"Invalid credentials" during notarization**: regenerate the app-specific password.
- **Notarization rejected**: download the log from `notarytool log <submission-id>`. Common causes are missing hardened runtime (already on by default in our project) or unsigned dylibs.
- **Changelog empty**: the workflow uses commits since the previous tag. If it's the first release, the changelog will be sparse.
